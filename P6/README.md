El programa está organizado de manera modular y el archivo principal se encarga de coordinar todo el proceso. Primero se lee el texto de entrada y se calcula la frecuencia de aparición de cada carácter, ya que esta información es fundamental para que el algoritmo de Huffman funcione correctamente. Con base en esas frecuencias, se construye el árbol de Huffman y posteriormente se generan los códigos binarios correspondientes para cada carácter. Finalmente, se calcula la tasa de compresión para comparar el tamaño del texto original con el tamaño del texto comprimido, lo que permite evaluar qué tan efectivo fue el algoritmo.
La lógica principal del algoritmo se encuentra en el archivo donde se implementa Huffman. Cada carácter se representa mediante un nodo que almacena su valor, su frecuencia y apuntadores a sus hijos. Para poder construir el árbol, es necesario combinar repetidamente los dos nodos con menor frecuencia, por lo que se mantiene una estructura ordenada que asegura que estos nodos siempre estén disponibles. En cada iteración, se toman los dos nodos menos frecuentes, se crea un nuevo nodo padre con la suma de sus frecuencias y se vuelve a insertar en la lista. Este proceso se repite hasta que solo queda un nodo, que se convierte en la raíz del árbol.
Una vez construido el árbol, se recorre de forma recursiva desde la raíz hasta las hojas para generar los códigos binarios. Cada vez que se avanza hacia la izquierda se agrega un cero y al avanzar hacia la derecha se agrega un uno. Cuando se llega a un nodo hoja, la secuencia acumulada de ceros y unos se guarda como el código comprimido del carácter correspondiente.
