El objetivo de esta práctica fue resolver la limitación que tienen los lenguajes de programación al trabajar con tipos de datos básicos como int o long, los cuales no permiten manejar números demasiado grandes. Para solucionar esto, implementé en lenguaje C y Python el Algoritmo de la Primaria, que básicamente simula el mismo procedimiento que usamos cuando multiplicamos números a mano en una hoja de papel.
Para lograrlo, en el código los números se manejan como arreglos de caracteres, es decir, como texto, lo que permite trabajar dígito por dígito sin importar qué tan largo sea el número. A partir de ahí, se calcula la longitud de cada número y se crea un arreglo para guardar el resultado, cuyo tamaño máximo es la suma de ambas longitudes. Este arreglo se inicializa en ceros para evitar errores causados por valores residuales en memoria.

La parte central del programa consiste en dos ciclos anidados que recorren los números de derecha a izquierda, tal como se hace en la multiplicación manual. En cada paso, los caracteres se convierten a valores numéricos y se multiplican entre sí, acumulando los resultados en la posición correspondiente del arreglo final. En esta etapa no se controla si el valor supera un solo dígito, ya que eso se corrige después.
Una vez terminadas todas las multiplicaciones, se realiza un segundo recorrido para manejar los acarreos. Si alguna posición tiene un valor mayor a 9, se deja únicamente la unidad y la decena se suma a la posición de la izquierda, asegurando que cada espacio del arreglo final contenga un solo dígito válido. Finalmente, el programa imprime el resultado, omitiendo los ceros a la izquierda para mostrar únicamente el número correcto al usuario.

Aunque este algoritmo tiene una complejidad cuadrática, resulta muy útil para comprender cómo se realizan las operaciones aritméticas a bajo nivel antes de pasar a algoritmos más eficientes. Con las pruebas realizadas, el programa funcionó correctamente, por ejemplo al multiplicar 12 por 34 y obtener como resultado 408.
